<Callout icon="terminal" color="#ff5d8f">
  The command group is `ticket`. The alias `tickets` also works. Default prefix is `,`.
</Callout>

All commands below require **Manage Server** (`manage_guild`).

---

## Overview

| Command | Description |
| ------- | ----------- |
| `,ticket` | Same as `,ticket overview` |
| `,ticket overview` | Dashboard: panels, forms, stats, log channels |

---

## Panels

| Command | Description |
| ------- | ----------- |
| `,ticket panel` | Open the interactive panel manager |
| `,ticket panel create (name)` | Create a panel |
| `,ticket panel delete (name)` | Delete a panel and its options |
| `,ticket panel send (name) [channel]` | Deploy panel to a channel |
| `,ticket panel mode (name) [button\|dropdown]` | Set or view display mode |
| `,ticket panel category (name) [category\|remove]` | Set or view ticket category |

**Examples:**

```javascript
,ticket panel create Support
,ticket panel send Support #open-ticket
,ticket panel mode Support dropdown
,ticket panel category Support Tickets
```

---

## Options

| Command | Description |
| ------- | ----------- |
| `,ticket option (panel)` | List options for a panel |
| `,ticket option create (panel) (label)` | Add a new option |
| `,ticket option delete (option_id)` | Remove an option by ID |

**Examples:**

```javascript
,ticket option Support
,ticket option create Support "Billing Question"
,ticket option delete 7
```

Use `,ticket option (panel)` to find option IDs before deleting.

---

## Forms

| Command | Description |
| ------- | ----------- |
| `,ticket form` | List all forms |
| `,ticket form create (name) (title)` | Create a form |
| `,ticket form delete (name)` | Delete a form |
| `,ticket form field (name)` | List fields on a form |
| `,ticket form field (name) add (label) [long]` | Add a field |
| `,ticket form field (name) delete (field_key)` | Remove a field |

**Examples:**

```javascript
,ticket form create Application "Staff Application"
,ticket form delete Application
```

See [Forms](/configuration/tickets/forms) for attaching forms to options and adding fields.

---

## Server configuration

| Command | Description |
| ------- | ----------- |
| `,ticket transcript [channel\|remove]` | Channel for ticket transcripts |
| `,ticket log [channel\|remove]` | Channel for ticket event logs |
| `,ticket staff [role\|remove]` | Global staff role for ticket permissions |

**Examples:**

```javascript
,ticket transcript #transcripts
,ticket transcript remove
```

```javascript
,ticket log #ticket-logs
,ticket log remove
```

```javascript
,ticket staff @Moderator
,ticket staff remove
```

---

## What is not a command

adore does **not** use prefix commands for day-to-day ticket actions like claiming or closing. Those happen through **buttons inside ticket channels**, configured per option in the panel manager.

There are also **no slash commands** for the ticket system — setup and management are prefix-only.

Members open tickets by clicking your deployed panel. Staff interact with tickets using channel buttons and normal Discord messaging.
