adore supports custom messages at the **panel** level and per **option**. Messages use adore's [embed syntax](/resources/scripting/embeds) plus ticket-specific variables.

---

## Where to edit messages

### Panel message

`,ticket panel` → select panel → **Message**

This is the embed/text shown above the ticket buttons or dropdown on your deployed panel.

Default if unset:

```javascript
{embed}
$v{title: Panel Name}
$v{description: Select an option below to create a ticket.}
```

### Option messages

`,ticket panel` → **Options** → select option → **Message**

| Message type | When it sends |
| ------------ | ------------- |
| **Greeting** | Ticket channel is created |
| **Close** | Ticket is closed |
| **Claim** | Staff claims the ticket |
| **Reopen** | Ticket is reopened |
| **Auto-Close** | Inactivity warning before auto-close |
| **Auto-Delete** | Warning before channel deletion |
| **Inactivity** | Inactivity notice |
| **Move** | Ticket moved to another category |
| **Required Roles** | Member lacks a required role |

Each message can be toggled on or off in the manager.

## Template variables

Use `{object.property}` in message templates. Conditionals use `{if:variable}text{/if}`.

### User (ticket creator)

| Variable | Description |
| -------- | ----------- |
| `{user.name}` | Username |
| `{user.display_name}` | Server nickname |
| `{user.id}` | User ID |
| `{user.mention}` | @mention |
| `{user.avatar}` | Avatar URL |

Aliases: `{member.*}`, `{creator.*}`

### Server

| Variable | Description |
| -------- | ----------- |
| `{guild.name}` | Server name |
| `{guild.id}` | Server ID |
| `{guild.membercount}` | Member count |

Alias: `{server.*}`

### Ticket

| Variable | Description |
| -------- | ----------- |
| `{ticket.id}` | Internal ticket ID |
| `{ticket.case_id}` | Case number shown in channel name |
| `{ticket.created}` | Open timestamp (UTC) |

### Option & panel

| Variable | Description |
| -------- | ----------- |
| `{option.label}` | Option label |
| `{option.id}` | Option ID |
| `{panel.name}` | Panel name |

### Staff actions

| Variable | Description |
| -------- | ----------- |
| `{staff.mention}` | Staff member involved |
| `{staff.name}` | Staff display name |
| `{claimer.mention}` | Who claimed |
| `{closer.mention}` | Who closed |
| `{reopener.mention}` | Who reopened |
| `{reason}` | Close/claim reason text |
| `{category}` | Category name after a move |
| `{time}` | Formatted time (auto-close/delete warnings) |
| `{roles}` | Required roles list |

### Form responses

| Variable | Description |
| -------- | ----------- |
| `{form.field_key}` | Value from a form field (use the field's key) |

## Example greeting

```javascript
{embed}
$v{title: Ticket #{ticket.case_id}}
$v{description: Welcome {user.mention}!

**Type:** {option.label}
**Opened:** {ticket.created}

A staff member will assist you shortly.}
$v{color: #6d84a0}
```

---

## Example close message

```javascript
{embed}
$v{title: Ticket Closed}
$v{description: Closed by {staff.mention}

{if:reason}**Reason:** {reason}{/if}

Thank you for contacting support!}
$v{color: #f87171}
```

---

## Conditionals

Show content only when a variable has a value:

```javascript
{if:reason}**Reason:** {reason}{/if}
{if:claimer.mention}Handled by {claimer.mention}{/if}
```

---

## Tips

- Test messages in a private channel before deploying to your support panel.
- Keep greeting messages concise — members already see form responses as a separate embed.
- Redeploy the panel after changing only the panel message; option messages apply to new tickets automatically.
