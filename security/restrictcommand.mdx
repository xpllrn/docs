## Overview

`,restrictcommand` lets you gate individual commands behind roles. Only members with an allowed role can run the command — useful alongside [fake permissions](/security/fake-permissions) when you want fine-grained access control.

<Info>
  Server owner and [fake owner](/security/fake-owner) bypass all restrictions.
</Info>

## Commands

<CodeGroup>

```javascript Syntax
,restrictcommand add (command) (role)
,restrictcommand remove (command) (role)
,restrictcommand list
,restrictcommand reset
```

```javascript Example
,restrictcommand add ban @Admin
,restrictcommand add kick @Moderator
,restrictcommand remove ban @Admin
```

</CodeGroup>

| Subcommand | Description |
| ---------- | ----------- |
| `add` | Restrict a command to a role |
| `remove` | Remove a role restriction from a command |
| `list` | View all restricted commands and their roles |
| `reset` | Remove all command restrictions |

<Warning>
  You cannot restrict the `restrictcommand` command itself.
</Warning>

## How it works

When a command is restricted, adore checks if the user has **any** of the roles assigned to that command. If not, the command is blocked.

Restrictions work with command aliases — if you alias `b` to `ban`, restricting `ban` also applies to the alias.

## Example setup

```javascript
,fakepermissions add @Moderator kick_members, moderate_members
```

```javascript
,restrictcommand add kick @Moderator
,restrictcommand add mute @Moderator
,restrictcommand add ban @Admin
```
